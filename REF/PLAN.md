# Glyph Hacker 实施计划

## 目标

实现 Android 端 Ingress Glyph Hack 辅助工具 MVP：

1. 持久悬浮窗控制开始/停止。
2. 通过 MediaProjection 每 300ms 采样录屏帧。
3. 用“节点-边图识别”识别 Glyph 序列（方案1）。
4. 检测到“开始绘制”模板后，通过 Accessibility 自动绘制。
5. 提供调试页面：导入录屏，实时叠加识别阶段与序列。
6. 在设置和架构中预留“手工序列匹配”（方案2）扩展位。

## 设计原则

- 当前主路径：`EDGE_SET`（边集合匹配）。
- 预留路径：`STROKE_SEQUENCE`（手工节点序列，先不实现核心逻辑）。
- 识别引擎在线/离线复用（录屏调试与实时识别同一套算法）。
- 参数全部可配置并持久化（DataStore）。

## 功能分解

### 1) 配置与状态

- DataStore 保存：
  - 识别模式（默认 `EDGE_SET`）
  - 采样间隔、亮度阈值、稳定帧数
  - 绘制速度（每边时长、字形间隔）
  - 空白截图 URI、开始绘制模板 URI
  - 节点标定结果与 ROI
- 运行状态总线：识别阶段、当前 glyph、已识别序列、置信度。

### 2) 节点标定与几何模型

- 从“空白截图”自动检测 11 个发光节点中心。
- 按固定拓扑为节点编号（0..10）。
- 生成候选边集合，并过滤穿越第三节点的非原子边。

### 3) 实时识别引擎

- 每帧对候选边采样亮度，判定激活边。
- 边集合匹配字典，得到 glyph 候选与置信度。
- 时序去抖：连续 N 帧一致才确认。
- 状态机：
  - `COMMAND_OPEN`
  - `GLYPH_DISPLAY`
  - `WAIT_GO`
  - `AUTO_DRAW`
  - `SCORING`

### 4) 模板触发与自动绘制

- 开始绘制模板匹配（可自定义）。
- 命中后按已识别序列触发 Accessibility 绘制。
- 方案1：由边集合自动求可绘制路径。
- 方案2 预留：未来可直接用手工节点序列。

### 5) 调试页面

- 导入本地录屏文件并按时间推进逐帧识别。
- 叠加显示：阶段、候选 glyph、已识别序列、激活边。
- 支持播放/暂停、逐帧、速度与阈值调参。

## 代码结构（初版）

- `MainActivity`：权限与设置入口。
- `OverlayControlService`：悬浮窗控制。
- `CaptureForegroundService`：录屏帧采样与识别。
- `GlyphAccessibilityService`：自动绘制。
- `glyph/*`：几何、标定、识别、状态机、模板匹配。
- `data/*`：设置与运行态。
- `ui/*`：设置页与调试页。

## 分阶段实施

### Milestone A

- Manifest 权限、服务声明。
- 设置持久化、基础 UI。
- 悬浮窗服务可启动。

### Milestone B

- 空白截图标定 11 点。
- 候选边可视化。

### Milestone C

- 实时录屏采样 + 边集合识别 + 序列聚合。
- GO 模板触发。

### Milestone D

- Accessibility 自动绘制与速度控制。
- 调试页面录屏叠加分析。

## 风险与对策

- 特效噪声导致误判：使用“中心-侧边差分亮度”+ 时序去抖。
- 分辨率差异：每设备配置节点标定并持久化。
- 手势兼容性：将每边时长与间隔开放可调。
- 阶段切换抖动：状态机引入稳定帧阈值和超时重置。
